交通事故严重度风险建模项目 Brief

本项目使用英国 DfT STATS19 道路事故数据，基于事故发生前可获得的道路、时间与环境信号， 对事故严重度进行三分类预测。它被设计为一个公开 GitHub 作品集项目：读者可以快速理解问题和交付物， 同时仍能看到数据治理、模型验证和研究边界。

研究模型Random Forest

1. 研究问题

交通安全分析不能只看事故数量，也需要关注哪些事故环境更可能对应高严重度后果。 本项目的问题是：在不使用明显后验信息的前提下，能否用道路、时间和环境特征建立一个可复现的事故严重度预测基线？

2. 数据与边界

  • 核心数据来自 UK DfT STATS19 碰撞、车辆与伤亡记录。
  • 伦敦天气数据只是覆盖有限的外部补充来源，不定义研究空间范围。
  • 默认使用 pre-event 特征，避免将事故结果后的信息混入模型。
  • 原始数据、处理后主表、训练模型和逐行误差文件不会上传到 GitHub。

3. 方法

项目比较 Logistic Regression、Random Forest 与 balanced HistGradientBoosting。 模型选择以 Macro F1 为主，因为事故严重度类别不均衡，单纯 Accuracy 容易掩盖 Fatal 与 Serious 等少数类风险。 同时单独报告 Fatal 召回率，用于讨论高成本漏判问题。

4. 可靠性证据

  • 分层 K 折验证用于检查重采样稳定性。
  • 时间外推切分模拟未来部署场景。
  • 空间分组外推用于检查地理迁移风险。
  • 泄漏检查用于确认高风险后验特征没有进入默认 pre-event 模型。
  • 校准和阈值分析用于解释 Fatal 筛查时概率与召回之间的取舍。

5. 主要局限

  • 当前项目是预测建模，不是因果推断。
  • 当前模型不是可直接部署的公共安全系统。
  • 天气外部补充覆盖有限，OSM、空气质量和 IMD 仍是后续扩展。
  • 模型仍需要更强的暴露量校正、空间验证和道路网络特征。

6. 后续研究路线

适合进一步扩展的方向包括：暴露量校正风险建模、道路网络分组验证、地理公平性误差分析、 更丰富的基础设施特征，以及在政策干预数据允许时引入因果识别设计。

7. 项目价值

对招聘者而言，本项目展示了从数据工程、建模、评估到公开发布的完整能力。 对研究场景而言，它展示了对数据泄漏、类别不平衡、时间/空间外推和模型局限的基本学术判断。
